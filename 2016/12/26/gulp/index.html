什么是gulp?

  • 之前使用grunt，比gulp麻烦
  • 前端自动化构建工具!
  • 基于nodejs
  • 重复的任务，可以通过gulp来完成
  • 如gulp转换less语法，js压缩,css压缩,html压缩!
  • gulp中的方法
    • task(定义一个任务)
    • src(匹配要处理的文件)
    • pipe(管道，控制任务内容的执行流程)
    • watch（监视文件变化，然后自动执行相应任务）
    • run(直接执行指定任务)
    • dest(指定文件的输出目录)

安装

  • 全局安装: npm install -g gulp-cli

项目代码结构

  • 开发环境代码
  • 生产环境代码
  • 比如: 在项目目录中的src中写代码,最终把压缩，处理后的代码生成到dist目录/public

gulp相关包

1. gulp-less

  • 转换less语法,需要用gulp-less包，配合gulp进行less语法转换
    • 在项目目录下安装gulp和gulp-less包，当然前提是在全局已经安装gulp了
    • npm install gulp --save-dev
    • npm install gulp-less --save-dev
    • 加-dev是因为这个包是用于开发的，用户不需要使用
  • 还需要在项目目录新建gulpfile.js (在这个文件中编写gulp相关的代码)

2. gulp-sass

  • npm install gulp-sass --save-devgulp也有转换sass文件的插件

3. gulp-uglify

  • 对js语法进行压缩与混淆!
  • npm install gulp --save-dev如果项目文件下已经安装过gulp就不需要安装gulp了
  • npm install gulp-uglify --save-dev 配合gulp对js代码进行压缩与混淆

4. gulp-htmlmin

  • 对html代码进行压缩
  • npm install gulp-htmlmin --save-dev 配合gulp对html进行压缩!
  • npm install gulp --save-dev
  • 相关文档(https://github.com/kangax/html-minifier)

5. gulp-cssnano

  • 对css进行压缩
  • npm install gulp-cssnano --save-dev// 对css进行压缩
  • npm install gulp --save-dev

gulp的常用方法：task,run,src,dest,watch

1. gulp.task

+ 作用：定义一个任务:
+ 示例:
+ 执行: 在命令行,gulpfile.js所在目录执行 `gulp 任务名`
  • default任务
  • 如果在命令行输入gulp 会直接执行当前目录的名为default的任务

2. gulp.run

  • run方法是用来执行其他任务
  • 参数是任务名,如果写成数组形式，那么数组中的元素就是要执行的任务名
  • 我们可以新建一个default任务，用这个任务去启动其他任务!

3. gulp.src

+ 作用: 匹配到我们想要处理的文件!
+ `gulp.src(['./*.js','!gulpfile.js'])`
+ 匹配当前目录下除了gulpfile.js的所有js文件
+ ！文件名.js 表示要排除的文件，即不执行它

  • 自动监视文件变化，自动刷新浏览器！

  • 切换到项目目录执行
