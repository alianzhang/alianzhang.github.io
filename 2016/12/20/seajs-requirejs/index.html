seajs和requirejs

模块化：seajs,requirejs

模块化的好处

  • 1.开发效率高
  • 2.可以利用
  • 3.便于后期维!

1.非模块化的方式!

  • 全局变量的问题:
    • 1.容易有冲突，会污染全局环境！
    • 2.js引用顺序很重要！(js文件引用存在先后顺序问题，和依赖关系)

2.以前的解决方式

2.1 命名空间!

  • 案例: jack,rose
  • 可以解决文件引用顺序问题!
  • 其实就是把原来数据，写到对象里了!

2.1 私有命名空间!

  • 案例: jack,rose
  • 可以解决，别人随便更改我们数据的问题！
    • 其实是通过调用方法，来返回一个变量(局部的，这样外部就不能够访问了)

2.3 封闭原则(千万不要改源码!)

  • 计算器!
  • 尽量不要改第三方库源码(不改 js 文件中的代码)
  • 但是我们可以扩展它的功能!

3 模块化规范!

  • 规范，只是一个约定,可以遵守，也可以不遵守!

  • 一个模块可能依赖于另外的模块

  • 如我们要造房子:
    • 1.如建造一座房子
      • 发现需要什么再去买什么
    • 2.对应的创建一个模块(可以把一个模块写到一个js文件中)
      • 一个a.js可以依赖于另一个b.js(可以在代码中表明依赖关系)
      • 我们代码在执行时，可以有两种方式：
  • CMD (前端模块化规范):
  • 【按需加载】a.js需要b.js时，我们就去加载b.js。如果a模块用到b,先加载a模块，再加载b模块
    • 对应 seajs// 进行模块化开发的库！我们用到时再加载！
    • 其实就是我们可以在 define 对应的 function 中任何位置使用 require()
    • 官网
    • 玉铂
  • AMD (前端模块化规范):
  • 【加载前置】,我a.js需要什么，就提前加载好。如果a模块用到b,先加载b模块，再加载a模块
    • 对应 RequireJS，先加载我们依赖的模块
    • 其实就是在我们 define 对应的 function 执行之前，帮我们引入模块
    • 官网

4 seajs

4.1 快速开始

  • 1.引包（引 seajs 文件夹下 dist 目录下的 sea.js）
  • 2.载入主模块如：(./main.js)
    • seajs.use('./main.js')
    • 第一个参数是对应模块文件名，后缀可以不用写
    • 当页面加载时，seajs 会帮我们自动拿到对应的文件./main.js
  • 3.定义主模块:(./main.js)
