jQuery 插件 lazyload.js 常用使用方法总结

延迟加载/懒加载

  • Lazy Load 这个 jQuery 插件，是用来缓冲加载图片的插件。
  • 这款插件在一开始不请求图片，它会检测你的滚动情况，当用户要访问时，才主动去请求要显示的图片。
  • 一开始不设置图片的src属性,当图片快要显示到可视区域时，再设置src属性!
  • 所以可以减少服务器的压力，避免不必要的资源下载。如果用户不看下面的图片，那加载全部的图片就造成了资源浪费。

Lazy Load 插件原理

修改目标 img 的 src 属性为 orginal 属性，从而中断图片的加载。检测滚动状态，然后把可视网页中的 img 的 src 属性还原，再加载图片，制造缓冲加载的效果。

存在的问题：在新版的浏览器中，即使你删除了 Javascript 控制的 src 属性，浏览器仍然会去加载这个图像。

解决的办法：直接修改 HTML 的结构，在 img 标签中添加新的属性，把 src 属性的值指向占位图片，添加 data-original 属性，让其指向真正的图像地址。

Lazy Load 的扩展

我们定义了这样的img结构，那么网页中，就不会加载源图像了。只有当 Javascript 执行，才会显示这个源图像。但是如果用户的浏览器不支持或者用户关掉了支持 Javascript 的选项，那么我们的这个图像就无法显示出来。也就是说，如果没有 Javascript 的支持，我们的图像就无法显示出来。

解决这个问题，我们需要引入 noscript 标签。
大体实现思路如下：用 noscript 包含真实的图像位置，当浏览器不支持 Javascript，直接显示图像。对现有图像，隐藏处理，使用 show() 方法触发显示。这样，如果浏览器不支持 Javascript，我们自定义的 img 就不会出现，而显示 noscript 里面的图像。
